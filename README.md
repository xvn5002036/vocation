# 正一法壇職官分發系統 (Taoist Divine Decree System)

這是一個結合道教傳統文化與現代網頁技術的應用程式。透過信士的生辰與法名，系統能依照《天壇玉格》與相關道教科儀邏輯，自動演算其職位歸屬、本命元帥以及專屬修持指引。

## 🌟 核心功能

* **年命干支演算**：輸入西元生年，系統自動精確換算農曆干支年命。
* **職銜自動分配**：根據不同的授籙等級（如：都功、盟威、五雷、靈寶等）分配對應的官署職位、委任狀與法權說明。
* **元帥歸屬邏輯**：結合天干五行與生肖北斗星君，匹配專屬護法元帥及其修持法門。
* **奏職寶誥生成**：自動生成包含壇、靖、治、炁等詳細資訊的傳統奏職疏文。
* **數位保存功能**：支持將分發結果生成精美的視覺化圖卡，便於保存與分享。

## 📂 系統關鍵檔案歸納

為了確保系統正常運作，以下為專案的核心檔案及其用途說明：

### 1. 核心邏輯
* **`src/App.jsx`**：應用的主要邏輯中心，負責干支計算、職位分發演算法及 UI 渲染。
* **`src/main.jsx`**：程式進入點，負責將 React 組件掛載至網頁渲染。

### 2. 環境配置
* **`package.json`**：定義專案依賴套件（如 React、Lucide-react）及執行指令。
* **`vite.config.js`**：Vite 工具的配置文件，處理開發伺服器與編譯設定。
* **`index.html`**：應用程式的基礎 HTML 模板。

### 3. 樣式配置
* **`tailwind.config.js`**：Tailwind CSS 的設定檔，定義了字體（Noto Serif TC）與道教專用配色。
* **`postcss.config.js`**：CSS 預處理器的配置檔案。
* **`src/index.css`**：全域樣式表，包含基礎 CSS 與捲軸自定義樣式。

### 4. 系統資料模組 (`src/constants/`)
* **`basic.js`**：存放五行屬性、天干對應與數理基礎。
* **`rank.js`**：定義各級授籙職銜與對應的官署職能。
* **`mapping.js`**：存放六十甲子治所、靖壇名稱與生肖星君映射表。
* **`marshals_meta.js`**：管理元帥的五行群組、天干歸屬。
* **`marshals_data.js`**：收錄 43 位元帥的詳細神蹟介紹與專屬修持指引。

## 📝 日後維護與資料擴充指南

### 1. 如何新增一位元帥？
若要加入新的元帥，您需要更新兩個檔案：
* **Step A: 更新 `marshals_data.js`**：加入新元帥的詳細名稱、介紹、修持法門。
* **Step B: 更新 `marshals_meta.js`**：
    1. 在 `MARSHAL_TITLES` 加入該元帥的職稱字典。
    2. 在 `ELEMENT_GROUP_META` 與 `STEM_MARSHALS` 中將其 ID 加入對應的五行或天干清單。

### 2. 如何修改職官或治所資料？
* **修改職稱/權限**：前往 `rank.js` 調整 `RANK_SYSTEM`。
* **修改治所/靖壇**：前往 `mapping.js` 找到對應的甲子年份進行修改。

### 3. 注意事項
* **保持導出 (Export)**：確保在 `constants/` 資料夾下的每個變數前都有 `export` 關鍵字。
* **引用路徑**：若移動檔案位置，請務必同步更新 `App.jsx` 頂部的 `import` 路徑。
